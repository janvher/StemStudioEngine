# Multiplayer

Guides for building real-time multiplayer experiences in StemStudio.

## Articles

1. [Multiplayer Overview](01-multiplayer-overview.md) — Room model, host authority, synchronized state, and setup
2. [Multiplayer Scripting](02-multiplayer-scripting.md) — Writing multiplayer-safe behaviors and state management

## Prerequisites

Read [Writing Behaviors](../scripting/02-writing-behaviors.md) first — multiplayer builds on top of the behavior system.

> **Audience layering**
> - **No-code creators:** flipping the multiplayer toggle in Project Settings is enough for most games — built-in behaviors are already multiplayer-safe. Read [Multiplayer Overview](01-multiplayer-overview.md) to understand the room model.
> - **JS developers:** [Multiplayer Scripting](02-multiplayer-scripting.md) covers host-authority guards, behavior data sync, and writing custom multiplayer-safe behaviors.
